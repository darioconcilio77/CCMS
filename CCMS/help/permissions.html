Permissions

D4P.CCMS.Permissions

CCMS ships four assignable permission sets. Assign one or more to each user or user group based on the access level required.

Permission sets

D4P BC ADMIN â€“ Full Access

Intended for administrators who manage customers, tenants, environments, and all operational features. This is the most powerful CCMS role.

  • RIMD (Read, Insert, Modify, Delete) on all CCMS data tables.
  • X (Execute) rights on all codeunits, pages, and reports.
Least privilege
Grant D4P BC ADMIN only to users who need to trigger administrative API operations (environment creation, session termination, updates, backups). For read-only access, use D4P BC ADMIN READ.

D4P BC ADMIN READ â€“ Read Only Access

Intended for users who need to view operational data without being able to modify configuration or trigger any API actions.

  • R (Read) on most CCMS data tables.
  • RIMD on KQL result tables (page executions, report executions, slow AL methods, extension lifecycle) â€” these are written during in-session telemetry queries.

D4P BC SETUP â€“ Setup Only

Grants access limited to the Setup table and page only. Suitable for users responsible solely for maintaining global settings (API URLs, number series, debug mode).

D4P BC TELEMETRY â€“ Telemetry Only

Grants access to all telemetry-related objects. Intended for users who run and analyse Application Insights KQL queries without access to environment administration.

  • RIMD on D4P AppInsights Connection and D4P KQL Query Store.
  • RIMD on all KQL result tables.
  • Execute on telemetry pages, codeunits, and the D4P Load Data report.

Assigning permissions

  1. Go to Users in BC and open the target user's card.
  2. Expand the User Permission Sets FastTab and add the relevant set.
    Use D4P BC ADMIN for full administrative access.
  3. Sign the user out and back in so the new permissions take effect.
User groups
All four permission sets are assignable via User Groups in BC. This is the recommended approach when managing permissions for multiple administrators.
