Sessions

D4P.CCMS.Session

The Session module gives administrators visibility into active user sessions running in a BC environment and provides the ability to terminate hanging or unwanted sessions remotely via the Admin Center API.

Session fields

FieldDescription
Session IDBC session identifier returned by the API.
User IDAAD Object ID or user name of the account that started the session.
Login DateDateTime when the session was started.
StatusCurrent session status (e.g. Active, Idle).
Client TypeType of BC client (e.g. WebClient, Background, ODataV4, NAS).

Actions

ActionAPI callDescription
Refresh GET .../environments/{name}/sessions Fetches the current list of active sessions from the Admin API and refreshes the session list. The data is a point-in-time snapshot.
Stop Session DELETE .../environments/{name}/sessions/{sessionId} Terminates the selected session after a confirmation prompt. The session is forcibly ended on the BC server.

How to stop a hanging session

  1. Open the Environment Card for the affected environment.
  2. Navigate to Sessions via the environment actions.
  3. Click Refresh to load the current session list from the API.
  4. Identify the session to terminate â€” look at the User ID, Login Date, and Client Type columns.
  5. Select the session row and click Stop Session. Confirm the prompt. The session is terminated immediately.
Impact of stopping a session
Terminating a session immediately disconnects the user without warning. Any unsaved work in that session will be lost. Use this action only for genuinely hanging or abandoned sessions.
Snapshot data
The session list is a snapshot taken at the time of the last Refresh. It does not update automatically. Refresh again to see the current state after stopping a session.
