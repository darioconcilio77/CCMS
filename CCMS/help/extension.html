Extensions

D4P.CCMS.Extension Table 62003

The Extension module manages installed apps across BC environments. It discovers which extensions are installed, checks for available updates, and can trigger the update process â€” all via the BC APIs without leaving BC.

Installed Apps

The D4P BC Installed App table stores a snapshot of extensions currently installed in each environment. Primary key: Customer No. + Tenant ID + Environment Name + App ID.

FieldDescription
App IDUnique GUID of the extension (App ID registered in AppSource or as PTE).
App NameDisplay name of the extension.
PublisherPublisher name.
VersionCurrently installed version (e.g. 24.5.0.0).
StateInstalled, Installing, or Uninstalling.
App TypePerTenant, Global, or Dev.
Available Update VersionLatest available version, populated after Get Available Updates.

Actions

ActionAPIDescription
Refresh Automation API GET .../extensions Calls the D4P Get Installed Apps report to fetch the full extension list from the Automation API and upserts records.
Get Available Updates Admin API GET .../availableupdates Fetches available update versions for each installed app and populates the Available Update Version field.
Update App Admin API POST .../update Schedules the selected app's update to the available version. An Operation record is created to track the async job.

Update workflow

  1. Open the Environment Card and go to the Installed Apps FactBox or navigate to Extensions.
  2. Click Refresh to get the current list of installed extensions.
  3. Click Get Available Updates to populate the Available Update Version column.
  4. Select the extension(s) you want to update.
  5. Click Update App. CCMS submits the update request to the Admin API and creates an Operation record. Monitor progress in Operations.
