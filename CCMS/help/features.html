Features

D4P.CCMS.Features

The Features module provides a UI to view and toggle BC feature flags for individual environments. BC environments expose optional and preview features that can be enabled or disabled via the Admin Center API.

Feature fields

FieldDescription
Feature IDUnique identifier of the feature from the Admin API (e.g. NewExperienceForFinancialReporting).
Feature NameHuman-readable display name of the feature.
StateEnabled, Disabled, or System (always on, cannot be toggled).
Data Update RequiredWhen true, enabling this feature may trigger a data upgrade, which can take additional time.

Feature states

StateMeaningToggleable?

Actions

ActionAPI callDescription
RefreshGET .../environments/{name}/featuresFetches the current feature list and states from the Admin API.
EnablePUT .../environments/{name}/features/{featureId}Enables the selected feature. A confirmation is shown if a data update is required.
DisablePUT .../environments/{name}/features/{featureId}Disables the selected feature.

How to enable or disable a feature

  1. Open the Environment Card for the target environment.
  2. Navigate to Features via the environment actions.
  3. Click Refresh to load the current feature states from the API.
  4. Select the feature you want to toggle.
  5. Click Enable or Disable. If the feature requires a data update, CCMS will warn you before proceeding.
Data updates
Features with Data Update Required = true may take longer to enable. During a data update, the environment may be temporarily unavailable to users. Schedule this action during a planned maintenance window.
System features
Features with State = System are enforced by Microsoft and cannot be toggled. The Enable and Disable actions are disabled for these features.
